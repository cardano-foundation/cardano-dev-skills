# Layout: Aiken + PyCardano (Stack 3)

Stack 3 targets a Python backend, data team, or scripting-heavy workflow. Aiken on-chain emits a CIP-57 `plutus.json` blueprint; PyCardano off-chain consumes it. The Python project uses Poetry for dependency management; pip-tools or uv are reasonable substitutions if your shop prefers them.

### Hello-world off-chain module -- `src/acme_offchain/blueprint.py`

```python
# src/acme_offchain/blueprint.py
#
# Loads the CIP-57 plutus.json blueprint emitted by `aiken build` and returns
# typed wrappers for each named validator. Every PyCardano caller imports the
# validator from here -- never re-derive the script hash.

import json
from dataclasses import dataclass
from pathlib import Path

from pycardano import PlutusV3Script, plutus_script_hash, ScriptHash

# Monorepo path: offchain/src/acme_offchain/blueprint.py -> ../../../onchain/plutus.json
# Single-repo path: src/acme_offchain/blueprint.py -> ../../plutus.json
_BLUEPRINT_PATH = Path(__file__).resolve().parents[3] / "onchain" / "plutus.json"


@dataclass(frozen=True)
class Validator:
    script: PlutusV3Script
    hash: ScriptHash


def get_validator(title: str) -> Validator:
    with _BLUEPRINT_PATH.open() as f:
        blueprint = json.load(f)
    for v in blueprint["validators"]:
        if v["title"] == title:
            script = PlutusV3Script(bytes.fromhex(v["compiledCode"]))
            return Validator(script=script, hash=plutus_script_hash(script))
    raise KeyError(
        f"Validator '{title}' not found in plutus.json. Did you run 'aiken build'?"
    )
```

### Hello-world off-chain module -- `src/acme_offchain/chain.py`

```python
# src/acme_offchain/chain.py
#
# Builds a PyCardano ChainContext. Devnet routes through Yaci Store (Blockfrost-
# compatible); preview/preprod/mainnet routes through real Blockfrost.
# Mainnet requires an explicit env var; the default is devnet.

import os
from pycardano import BlockFrostChainContext, Network


def get_context() -> BlockFrostChainContext:
    network_name = os.environ.get("CARDANO_NETWORK", "devnet")

    if network_name == "devnet":
        base_url = os.environ.get("YACI_STORE_URL", "http://localhost:10000")
        # Yaci Store exposes a Blockfrost-compatible API; project_id is unused
        # but PyCardano requires a non-empty string.
        return BlockFrostChainContext(
            project_id="devnet",
            base_url=base_url + "/api/v1",
            network=Network.TESTNET,
        )

    project_id = os.environ.get("BLOCKFROST_PROJECT_ID")
    if not project_id:
        raise RuntimeError(
            "BLOCKFROST_PROJECT_ID required when CARDANO_NETWORK is not devnet"
        )

    if network_name == "mainnet":
        return BlockFrostChainContext(
            project_id=project_id,
            base_url="https://cardano-mainnet.blockfrost.io/api/v0",
            network=Network.MAINNET,
        )

    base_url = {
        "preview": "https://cardano-preview.blockfrost.io/api/v0",
        "preprod": "https://cardano-preprod.blockfrost.io/api/v0",
    }[network_name]
    return BlockFrostChainContext(
        project_id=project_id, base_url=base_url, network=Network.TESTNET
    )
```

The full version of `lock.py` derives the dev wallet from the mnemonic, computes the script address, builds a `TransactionOutput` to that address with the encoded `HelloDatum`, lets `TransactionBuilder` select inputs and fees, signs with the payment signing key, and submits via the chain context. A matching `redeem.py` queries the script address via the chain context, picks the locked UTxO, attaches the validator script + redeemer + datum to the input, includes a collateral UTxO, and signs as usual. Hand off to `build-transaction` for the complete pattern.

## Network config patterns

Three places to flip when switching networks:

1. `CARDANO_NETWORK` in `.env` (devnet | preview | preprod | mainnet). Default to a testnet; never mainnet.
2. `chain.py` (already shown above) returns a `BlockFrostChainContext` configured for the active network. The `Network.TESTNET` / `Network.MAINNET` argument matches the CARDANO_NETWORK env.
3. If you scaffold a sibling frontend (see below), its `.env.local` mirrors `CARDANO_NETWORK` and uses its own Blockfrost project ID.

Each non-devnet network needs its own Blockfrost project ID from https://blockfrost.io (distinct prefixes: `preview...`, `preprod...`, `mainnet...`). Faucets for preview and preprod live at https://docs.cardano.org/cardano-testnets/tools/faucet. Yaci DevKit ships its own faucet via `yaci-cli faucet send`.

## Frontend (optional sibling, default ON in the scaffold)

A Python backend cannot run in the browser. When the developer opts in to a frontend, scaffold a separate Next.js App Router app вЂ” independent of the Python backend вЂ” that talks directly to Blockfrost from the user's browser. The frontend reads the same `onchain/plutus.json` (committed alongside the backend) and does its own client-side tx building.
