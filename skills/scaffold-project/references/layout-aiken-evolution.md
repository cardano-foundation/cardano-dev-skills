# Layout: Aiken + Evolution SDK (Stack 1)

Stack 1 targets a TypeScript team that wants a pure-JS off-chain toolchain with composable transaction shapes. Aiken on-chain emits a CIP-57 `plutus.json` blueprint; Evolution SDK off-chain consumes it.

Evolution wraps the Effect library internally — the public API hides most of it, but you may occasionally see Effect types in advanced usage. See https://effect.website/ if you want to go deeper. The patterns below stay close to standard async/await to keep the scaffold approachable.

### Hello-world off-chain script -- `offchain/src/blueprint.ts`

```typescript
// offchain/src/blueprint.ts
//
// Loads the CIP-57 plutus.json blueprint and exposes typed validator entries.
// Identical in spirit to stack 1; Evolution consumes the same hex CBOR.

import { readFileSync } from "node:fs";
import { fileURLToPath } from "node:url";
import { dirname, resolve } from "node:path";

const here = dirname(fileURLToPath(import.meta.url));
const blueprintPath = resolve(here, "../../onchain/plutus.json");
//                                ^^^^^^^^^^^^^^^^^^^^^^^^^^
// Monorepo path. For single-repo, change to "../plutus.json".

type Blueprint = {
  validators: Array<{
    title: string;
    compiledCode: string;
    hash: string;
  }>;
};

const blueprint = JSON.parse(readFileSync(blueprintPath, "utf-8")) as Blueprint;

export function getValidator(title: string) {
  const v = blueprint.validators.find((x) => x.title === title);
  if (!v) {
    throw new Error(
      `Validator '${title}' not found in plutus.json. Did you run 'aiken build'?`,
    );
  }
  return { script: v.compiledCode, hash: v.hash };
}
```

### Hello-world off-chain script -- `offchain/src/client.ts`

```typescript
// offchain/src/client.ts
//
// Configures an Evolution client. Devnet uses a Blockfrost-compatible Yaci
// Store endpoint; preview/preprod/mainnet uses a real Blockfrost project.
// Search ${CLAUDE_SKILL_DIR}/../../docs/sources/evolution-sdk/ for the latest
// provider configuration shapes; the names below may evolve.

import "dotenv/config";
import { preprod, preview, mainnet, client as makeClient } from "@evolution-sdk/evolution";

export function getClient() {
  const network = process.env.CARDANO_NETWORK ?? "devnet";

  // Devnet: point at Yaci Store's Blockfrost-compatible API.
  if (network === "devnet") {
    const baseUrl = (process.env.YACI_STORE_URL ?? "http://localhost:10000") + "/api/v1";
    return makeClient(preprod)              // preprod params are a reasonable default
      .withBlockfrost({ baseUrl, projectId: "devnet" })
      .withSeed({ mnemonic: process.env.DEV_WALLET_MNEMONIC!, accountIndex: 0 });
  }

  const projectId = process.env.BLOCKFROST_PROJECT_ID;
  if (!projectId) {
    throw new Error("BLOCKFROST_PROJECT_ID required when CARDANO_NETWORK is not devnet");
  }
  const netConfig = network === "mainnet" ? mainnet : network === "preview" ? preview : preprod;
  const baseUrl =
    network === "mainnet"
      ? "https://cardano-mainnet.blockfrost.io/api/v0"
      : network === "preview"
        ? "https://cardano-preview.blockfrost.io/api/v0"
        : "https://cardano-preprod.blockfrost.io/api/v0";

  return makeClient(netConfig)
    .withBlockfrost({ baseUrl, projectId })
    .withSeed({ mnemonic: process.env.DEV_WALLET_MNEMONIC!, accountIndex: 0 });
}
```

### Hello-world off-chain script -- `offchain/src/hello/lock.ts`

```typescript
// offchain/src/hello/lock.ts
//
// Locks 5 ADA at the hello validator. Run with: npm run tx:lock

import "dotenv/config";
import { Address, Assets, Data } from "@evolution-sdk/evolution";
import { getClient } from "../client.js";
import { getValidator } from "../blueprint.js";

async function main() {
  const client = getClient();
  const hello = getValidator("hello.hello.spend");

  // Build a script address from the compiled validator. Helper shape may vary
  // by Evolution version; check the bundled docs for the current API.
  const scriptAddr = await client.scripts.toAddress({
    type: "PlutusV3",
    script: hello.script,
  });

  const ownerHash = await client.wallet.getPaymentKeyHash();
  // Datum: { owner: VerificationKeyHash }. Encode according to your Aiken type.
  const datum = Data.to({ owner: ownerHash });

  const tx = await client
    .newTx()
    .payToContract({
      address: Address.fromBech32(scriptAddr),
      datum: { kind: "inline", value: datum },
      assets: Assets.fromLovelace(5_000_000n),
    })
    .build();

  const signed = await tx.sign();
  const txHash = await signed.submit();
  console.log("Locked. Tx hash:", txHash);
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

A matching `redeem.ts` queries the script address, picks the locked UTxO, attaches the validator (or a reference script), and spends back to the wallet. Hand off to `build-transaction` for the full pattern.

## Notes on the Evolution API

The exact method names above (`payToContract`, `scripts.toAddress`, `Assets.fromLovelace`) reflect the public API at the time of writing. Evolution is actively evolving; before pasting these into production code, verify against the current bundled docs:

- `${CLAUDE_SKILL_DIR}/../../docs/sources/evolution-sdk/` -- top-level guides
- `${CLAUDE_SKILL_DIR}/../../docs/sources/evolution-sdk-packages/` -- per-package READMEs with exact signatures

## Network config patterns

Three places to flip when switching networks:

1. `CARDANO_NETWORK` in `.env` (devnet | preview | preprod | mainnet). The default is a testnet; never mainnet.
2. The Evolution client factory in `client.ts` (already shown above) switches the Blockfrost base URL and network config.
3. Browser code, if you have a frontend, reads `NEXT_PUBLIC_NETWORK` and uses a Next.js API route to keep the project ID server-side. See the frontend section below.

For each non-devnet network you target, paste a project ID from https://blockfrost.io into `BLOCKFROST_PROJECT_ID`. Each network has a separate key prefix on Blockfrost (`preview...`, `preprod...`, `mainnet...`); you cannot reuse a preview key for preprod. Faucets for preview and preprod live at https://docs.cardano.org/cardano-testnets/tools/faucet (the page selector picks the right network). Yaci DevKit has its own built-in faucet via `yaci-cli faucet send`.

## Frontend (optional, default ON in the scaffold)

When the developer opts in, scaffold a Next.js App Router app under `frontend/` (same monorepo as off-chain). It does its own tx building client-side and talks to Blockfrost via a small server-side Next.js API route. Wallet integration is via Evolution's CIP-30 bridge.

`frontend/package.json` essentials:

```jsonc
{

  },

    "@evolution-sdk/lucid": "^X.Y.Z",            // PIN; same major as the off-chain version

    "@types/node": "^20.0.0",

    "@types/react-dom": "^18.3.0",

```

`frontend/app/page.tsx` (sketch):

```tsx
"use client";
import { useState } from "react";
import { connectAndLock } from "./lib/tx";

export default function Page() {
  const [hash, setHash] = useState<string | null>(null);
  return (
    <main>
      <h1>Hello, Evolution + CIP-30</h1>
      <button onClick={async () => setHash(await connectAndLock())}>
        Connect wallet and lock 5 ADA
      </button>
      {hash && <p>tx: {hash}</p>}
    </main>
  );
}
```

CIP-30 details: hand off to `connect-wallet`. The pattern is the standard one — discover wallets on `window.cardano`, call `enable()`, hand the API to Evolution via `selectWallet.fromAPI(walletApi)`, then build and sign transactions the same way the backend code does.

Keep `BLOCKFROST_PROJECT_ID` out of `NEXT_PUBLIC_*` env. Front Blockfrost calls behind a Next.js route handler so the project ID stays server-side. Wallet calls remain client-side because there is no server-side wallet.
