# Layout: Aiken + cardano-client-lib (Stack 4)

Stack 4 targets a JVM team -- Java, Kotlin, or any other JVM language. Aiken on-chain emits a CIP-57 `plutus.json` blueprint; cardano-client-lib off-chain consumes it. The Java project uses Maven; Gradle is a reasonable substitution if your team prefers it (translate the `pom.xml` to a `build.gradle.kts` 1:1).

### Hello-world off-chain class -- `Blueprint.java`

```java
// src/main/java/org/acme/offchain/Blueprint.java
//
// Loads the CIP-57 plutus.json blueprint emitted by `aiken build` and returns
// typed wrappers for each named validator. Every cclib caller asks Blueprint
// for the script -- never re-derive the script hash by hand.

package org.acme.offchain;

import com.bloxbean.cardano.client.plutus.spec.PlutusV3Script;
import com.bloxbean.cardano.client.util.HexUtil;
import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;

import java.nio.file.Files;
import java.nio.file.Path;

public final class Blueprint {

    public record Validator(PlutusV3Script script, String scriptHash) {}

    private final JsonNode root;

    public Blueprint(Path plutusJsonPath) throws Exception {
        var mapper = new ObjectMapper();
        this.root = mapper.readTree(Files.readString(plutusJsonPath));
    }

    public Validator getValidator(String title) {
        for (JsonNode v : root.get("validators")) {
            if (title.equals(v.get("title").asText())) {
                var cbor = HexUtil.decodeHexString(v.get("compiledCode").asText());
                var script = PlutusV3Script.builder().cborHex(v.get("compiledCode").asText()).build();
                return new Validator(script, v.get("hash").asText());
            }
        }
        throw new IllegalArgumentException(
            "Validator '" + title + "' not found in plutus.json. Did you run 'aiken build'?"
        );
    }
}
```

### Hello-world off-chain class -- `ChainClient.java`

```java
// src/main/java/org/acme/offchain/ChainClient.java
//
// Builds a BackendService. Devnet uses Yaci Store's Blockfrost-compatible API;
// preview/preprod/mainnet uses real Blockfrost. Mainnet requires explicit opt-in.

package org.acme.offchain;

import com.bloxbean.cardano.client.backend.api.BackendService;
import com.bloxbean.cardano.client.backend.blockfrost.service.BFBackendService;
import com.bloxbean.cardano.client.common.model.Networks;
import com.bloxbean.cardano.client.common.model.Network;

public final class ChainClient {

    public record ConfiguredBackend(BackendService backend, Network network) {}

    public static ConfiguredBackend fromEnv() {
        String network = System.getenv().getOrDefault("CARDANO_NETWORK", "devnet");

        if ("devnet".equals(network)) {
            String url = System.getenv().getOrDefault("YACI_STORE_URL", "http://localhost:10000");
            // Yaci Store exposes a Blockfrost-compatible API on /api/v1.
            var backend = new BFBackendService(url + "/api/v1/", "devnet");
            return new ConfiguredBackend(backend, Networks.testnet());
        }

        String projectId = System.getenv("BLOCKFROST_PROJECT_ID");
        if (projectId == null || projectId.isBlank()) {
            throw new IllegalStateException(
                "BLOCKFROST_PROJECT_ID required when CARDANO_NETWORK is not devnet"
            );
        }
        return switch (network) {
            case "mainnet" -> new ConfiguredBackend(
                new BFBackendService(
                    "https://cardano-mainnet.blockfrost.io/api/v0/", projectId),
                Networks.mainnet());
            case "preview" -> new ConfiguredBackend(
                new BFBackendService(
                    "https://cardano-preview.blockfrost.io/api/v0/", projectId),
                Networks.preview());
            case "preprod" -> new ConfiguredBackend(
                new BFBackendService(
                    "https://cardano-preprod.blockfrost.io/api/v0/", projectId),
                Networks.preprod());
            default -> throw new IllegalArgumentException("Unknown network: " + network);
        };
    }
}
```

### Hello-world off-chain class -- `Lock.java`

```java
// src/main/java/org/acme/offchain/hello/Lock.java
//
// Locks 5 ADA at the hello validator. Run with:
//   mvn -q exec:java -Dexec.mainClass=org.acme.offchain.hello.Lock

package org.acme.offchain.hello;

import com.bloxbean.cardano.client.account.Account;
import com.bloxbean.cardano.client.address.AddressProvider;
import com.bloxbean.cardano.client.api.model.Amount;
import com.bloxbean.cardano.client.function.helper.SignerProviders;
import com.bloxbean.cardano.client.plutus.spec.PlutusV3Script;
import com.bloxbean.cardano.client.quicktx.QuickTxBuilder;
import com.bloxbean.cardano.client.quicktx.Tx;

import org.acme.offchain.Blueprint;
import org.acme.offchain.ChainClient;

import java.nio.file.Path;

public final class Lock {

    public static void main(String[] args) throws Exception {
        var configured = ChainClient.fromEnv();
        var blueprintPath = Path.of(System.getenv().getOrDefault(
            "BLUEPRINT_PATH", "../onchain/plutus.json"));
        var blueprint = new Blueprint(blueprintPath);
        var hello = blueprint.getValidator("hello.hello.spend");

        // Dev wallet from mnemonic; treat as dev-only.
        var mnemonic = System.getenv("DEV_WALLET_MNEMONIC");
        if (mnemonic == null) throw new IllegalStateException("DEV_WALLET_MNEMONIC not set");
        var account = new Account(configured.network(), mnemonic);

        // Script address.
        var scriptAddr = AddressProvider.getEntAddress(hello.script(), configured.network());

        // Build, sign, submit using QuickTxBuilder.
        var quickTxBuilder = new QuickTxBuilder(configured.backend());
        var tx = new Tx()
            .payToContract(scriptAddr.toBech32(), Amount.lovelace(5_000_000L).build(),
                // Inline datum: { owner: <pubkey hash bytes> }. Encode using cclib's
                // PlutusData builders; see ${CLAUDE_SKILL_DIR}/../../docs/sources/cardano-client-lib/.
                /* datum */ null
            )
            .from(account.baseAddress());

        var result = quickTxBuilder
            .compose(tx)
            .withSigner(SignerProviders.signerFrom(account))
            .completeAndWait();

        System.out.println("Locked. Tx hash: " + result.getValue());
    }
}
```

The datum placeholder above is intentional: encoding `HelloDatum { owner: bytes }` via cclib's `PlutusData` API has a few possible shapes depending on the cclib version. Consult `${CLAUDE_SKILL_DIR}/../../docs/sources/cardano-client-lib/` for the current `PlutusData.builder()` and `ConstrPlutusData` patterns before pasting the encoding into production code.

A matching `Redeem.java` queries the script address through the backend service, picks the locked UTxO, attaches the validator + redeemer + datum to the spending input, sets a collateral UTxO, and submits. Hand off to `build-transaction` for the complete pattern.

## Network config patterns

Three places to flip when switching networks:

1. `CARDANO_NETWORK` in `.env` (devnet | preview | preprod | mainnet). Default to a testnet; never mainnet.
2. `ChainClient.java` (already shown above) builds a `BFBackendService` with the matching Blockfrost URL and a `Networks.testnet() / preview() / preprod() / mainnet()` selector.
3. If you scaffold a sibling frontend (see below), its `.env.local` mirrors `CARDANO_NETWORK` and uses its own Blockfrost project ID.

Each non-devnet network needs its own Blockfrost project ID from https://blockfrost.io (distinct prefixes: `preview...`, `preprod...`, `mainnet...`). Faucets for preview and preprod live at https://docs.cardano.org/cardano-testnets/tools/faucet. Yaci DevKit ships its own faucet via `yaci-cli faucet send`.

## Frontend (optional sibling, default ON in the scaffold)

A JVM backend cannot run in the browser. When the developer opts in to a frontend, scaffold a separate Next.js App Router app вЂ” independent of the Java backend вЂ” that talks directly to Blockfrost from the user's browser. The frontend reads the same `onchain/plutus.json` (committed alongside the backend) and does its own client-side tx building.
